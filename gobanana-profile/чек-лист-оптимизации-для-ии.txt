ЧЕК-ЛИСТ ОПТИМИЗАЦИИ САЙТА ДЛЯ ИИ-АГЕНТОВ (AEO 2026)

Цель: сделать так, чтобы ChatGPT, Perplexity и YandexGPT рекомендовали именно вас.

🔥 КРИТИЧНО ВАЖНО (срок: 1-5 дней)

1. robots.txt для ИИ-ботов (1 день)
   - Аудит текущего robots.txt на предмет блокировки AI-ботов
   - Добавление разрешений для краулеров нейросетей
   - Проверка, что ключевые страницы не закрыты от индексации
   - Настройка отдельных правил для каждого AI-бота
   - Мониторинг логов сервера на посещения AI-краулеров

   Код для robots.txt:
   ```
   # Разрешение доступа ИИ-краулерам
   User-agent: GPTBot
   Allow: /
   User-agent: OAI-SearchBot
   Allow: /
   User-agent: ChatGPT-User
   Allow: /
   User-agent: ClaudeBot
   Allow: /
   User-agent: PerplexityBot
   Allow: /
   User-agent: Applebot-Extended
   Allow: /
   User-agent: Meta-ExternalAgent
   Allow: /
   User-agent: Amazonbot
   Allow: /
   User-agent: cohere-ai
   Allow: /
   User-agent: Bytespider
   Allow: /
   User-agent: CCBot
   Allow: /
   User-agent: Google-Extended
   Allow: /
   ```

2. Schema.org разметка (3-5 дней)
   - Organization/HomeBuilder Schema на ВСЕ страницы
   - FAQPage Schema на страницу FAQ
   - QAPage Schema для страниц с вопросами-ответами
   - Article/BlogPosting Schema для контентных страниц
   - HowTo Schema для этапов работы
   - Объединение через @graph для повышения AI-цитирований

   Organization Schema пример:
   ```json
   <script type="application/ld+json">
   {
     "@context": "https://schema.org",
     "@type": "HomeBuilder",
     "name": "Название компании",
     "description": "Строительство загородных домов под ключ",
     "url": "https://site.ru",
     "telephone": "+7 (XXX) XXX-XX-XX",
     "address": {
       "@type": "PostalAddress",
       "streetAddress": "ул. Примерная, д. 1",
       "addressLocality": "Город",
       "addressRegion": "Область",
       "addressCountry": "RU"
     },
     "sameAs": [
       "https://vk.com/company",
       "https://t.me/company"
     ]
   }
   </script>
   ```

⚠️ ВАЖНО (срок: 1-2 недели)

3. FAQ-раздел с 30+ вопросами (1-2 нед.)
   - Сбор 30-50 реальных вопросов из CRM и звонков
   - Прямые, содержательные ответы: 50-150 слов каждый
   - Группировка: цены, сроки, гарантии, технологии, процесс
   - FAQPage Schema разметка
   - Обновление по Яндекс.Вордстат ежемесячно
   - Вопросы в разговорном стиле: «Сколько стоит?», не «Стоимость»

4. QAPage + Article Schema (2-3 дня)
   - QAPage для страниц с одним вопросом = развёрнутый ответ
   - Указание answerCount и upvoteCount для сигналов доверия
   - Article Schema с author и credentials для E-E-A-T
   - datePublished + dateModified для свежести контента

5. Мета-теги даты и авторства (2-3 дня)
   - Дата публикации и обновления на всех страницах
   - Указание авторства (E-E-A-T сигналы)
   - Текущий год в title и description ключевых страниц
   - Speakable Schema для голосовых ассистентов (на вырост)

   Пример мета-тегов:
   ```html
   <meta property="article:published_time" content="2025-01-15T10:00:00+03:00">
   <meta property="article:modified_time" content="2026-02-01T12:00:00+03:00">
   <meta property="article:author" content="Имя, должность">
   <meta property="og:title" content="Дома под ключ — от 2,5 млн [2026]">
   ```

💡 ПОЛЕЗНО (срок: 2 недели - 2 месяца)

6. Answer-first структура контента (2-3 нед.)
   - Прямой ответ / УТП в первых 50-100 словах
   - Строгая иерархия: H1 → H2 → H3
   - HTML-таблицы для сравнений
   - Списки <ul>/<ol> вместо текстовых блоков
   - Alt-тексты для ВСЕХ изображений
   - Один абзац = один тезис

   Пример:
   ```
   ❌ ДО: "Добро пожаловать на наш сайт! Мы — команда профессионалов..."
   ✅ ПОСЛЕ: "Строим каркасные, брусовые и газобетонные дома от 2,5 млн руб. Срок — от 2 мес. Гарантия 10 лет. 500+ сданных объектов."
   ```

7. Файл llms.txt в корне сайта (1 день) - экспериментальный
   - Составление списка приоритетных страниц
   - Краткое описание компании в формате Markdown
   - Структурирование ссылок по разделам
   - Размещение по адресу yoursite.ru/llms.txt

8. Тематические кластеры (1-2 мес.)
   - Hub-страницы по каждой технологии
   - 5-10 детальных статей-сателлитов на каждый hub
   - Перелинковка кластеров между собой
   - Покрытие «дерева вопросов» — от общих к частным

9. Скорость загрузки для AI-краулеров
   - Загрузка до 2-3 секунд
   - Server-side rendering (SSR) - нейросети не выполняют JS
   - Минимизация блокирующих ресурсов
   - CDN для быстрой отдачи контента

   Проверка доступности для ботов:
   ```bash
   curl -A "GPTBot" https://site.ru/ | head -100
   grep -E "(GPTBot|ClaudeBot|PerplexityBot)" /var/log/nginx/access.log
   ```

10. Мониторинг AI Share of Voice (постоянно)
    - Еженедельное тестирование в ChatGPT, Perplexity, YandexGPT
    - Отслеживание упоминаний бренда в ответах нейросетей
    - Сравнение с конкурентами по частоте упоминаний

    Запросы для тестирования (ежемесячно):
    1. «Какую компанию выбрать для строительства дома в [Регион]?»
    2. «Сколько стоит построить дом под ключ в [Регион]?»
    3. «Лучшие строительные компании [Регион] рейтинг»
    4. «Каркасный или газобетонный дом — что выбрать?»
    5. «Отзывы о строительной компании [Название]»

11. Экспертный контент для цитирования
    - Уникальные данные: собственная статистика, реальные кейсы
    - Числовые факты в каждой статье (сроки, цены, площади)
    - Таблицы сравнения — нейросети извлекают табличные данные лучше
    - Оригинальные исследования и обзоры рынка
    - Обновление ключевых страниц минимум раз в квартал

📊 ОЖИДАЕМЫЕ РЕЗУЛЬТАТЫ
- ×2.8 рост цитирований в ИИ-ответах (данные AirOps, 2025)
- 83% AI-цитат по коммерческим запросам — страницы, обновлённые за последние 12 мес.
- 60%+ цитат — страницы, обновлённые за последние 6 мес.

- ChatGPT: разговорный стиль, полнота раскрытия
